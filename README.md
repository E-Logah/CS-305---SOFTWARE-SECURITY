# CS-305---SOFTWARE-SECURITY
## Artemis Financial Summary
Artemis Financial is a financial services company that required improvements to the security of its software application. The client wanted to ensure that customer data and financial information were protected during transmission and storage. The project focused on identifying vulnerabilities within the application, implementing secure coding practices, and strengthening the overall security posture of the software.
## What I Did Well
One area I performed well was identifying software vulnerabilities and evaluating potential security risks. I successfully used dependency analysis tools and manual code review techniques to locate weaknesses within the application. I also implemented security enhancements such as secure communications, encryption, and dependency management to reduce risk.
## Challenging or Helpful Aspects
The most challenging aspect of the vulnerability assessment process was interpreting the results of automated scanning tools and determining which vulnerabilities required remediation. At the same time, this was one of the most valuable learning experiences because it demonstrated how security professionals prioritize risks and validate findings before implementing fixes.
## Increasing Layers of Security
I increased security by implementing secure communications using SSL/TLS, updating vulnerable dependencies, validating software components, and applying secure coding practices. In future projects, I would continue using vulnerability scanning tools, code reviews, dependency analysis, and risk assessments to identify and mitigate security threats.
## Ensuring Functionality and Security
After making security improvements, I tested the application to verify that all functionality remained intact. I also reran dependency-check reports and reviewed application behavior to confirm that no new vulnerabilities were introduced during refactoring. This process helped ensure both security and functionality were maintained.
## Resources, Tools, and Practices
Throughout this project I used Eclipse, Maven, OWASP Dependency-Check, Java security libraries, secure coding practices, and software testing techniques. These resources provided practical experience that will be valuable in future software development and cybersecurity roles.
## Value to Future Employers
This project demonstrates my ability to analyze software vulnerabilities, implement security controls, perform risk assessments, and validate secure software solutions. Future employers can use this work as evidence of my understanding of secure software development practices and cybersecurity principles.
